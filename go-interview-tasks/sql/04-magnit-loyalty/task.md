# 04. Магнит — программа лояльности и покупки

**Контекст:** ритейл, карта лояльности, начисление бонусов (Магнит, похожие сценарии в X5 / Ленте). На собесе — **схема баллов** и аналитика по покупкам.

**Время:** ~30 мин.

## Дано

`schema.sql`, `seed.sql`.

- `customers` — покупатель
- `loyalty_cards` — карта, `points_balance`, уровень `tier` (`base` / `silver` / `gold`)
- `stores` — магазин (`format`: `magnit`, `cosmetic`, `pharmacy`)
- `purchases` — чек: сумма в копейках, дата
- `point_ledger` — журнал начислений/списаний (`delta`, `reason`)

Баланс на карте должен совпадать с `SUM(delta)` по журналу (на собесе — как обеспечить).

## Часть 1. Проектирование

Опиши в `design.md`:

1. Как начислять **1%** от `total_kopecks` покупки в баллы (1 балл = 1 копейка).
2. Как списывать баллы при оплате части чека без **двойного списания**.
3. Нужна ли идемпотентность по `purchase_id` в `point_ledger`.

## Часть 2. Запросы

As-of: `2025-06-15`.

### 4.1. Топ покупателей

Топ-5 клиентов по сумме покупок за **90 дней** (неотменённые чеки; отмены в данных нет — считаем все `purchases`).

Колонки: `customer_id`, `customer_name`, `spend_kopecks`, `points_balance`, `tier`.

### 4.2. «Спящие» с баллами

Клиенты, у которых:

- `points_balance >= 5000`;
- последняя покупка **раньше** чем `2025-06-15 - 60 days`;
- хотя бы одна покупка в истории.

Колонки: `customer_id`, `customer_name`, `points_balance`, `last_purchase_date`, `days_since_purchase`.

Сортировка: по убыванию `points_balance`.

### 4.3. Магазины по формату

За последние **30 дней**: по каждому `format` — число чеков и средний чек в копейках (целое, `ROUND`).

Колонки: `format`, `receipts_count`, `avg_check_kopecks`.

## На что смотрят

- журнал проводок vs mutable balance
- оконные функции / `MAX(created_at)` vs `LATERAL`
- аккуратность с датами и `GROUP BY`
